
# M6-B 键盘改键工具 - 程序员交接工作说明书

---

## 1. 项目概述

### 1.1 项目名称
**M6-B 键盘改键工具** (Keyboard Remap Tool for RAMA WORKS M6-B)

### 1.2 项目目的
为 RAMA WORKS M6-B 键盘提供基于 WebHID 的图形化改键工具，支持：
- 键位映射修改
- 宏功能设置（M0-M15）
- 灯光控制（背光/RGB）
- EEPROM 读写

### 1.3 技术栈
- **前端**: HTML5 + CSS3 + JavaScript (ES6+)
- **通信**: WebHID API (Chrome/Edge 89+)
- **协议**: VIA/QMK 协议
- **服务器**: Python SimpleHTTPServer

## 3. 核心功能模块

### 3.1 键位映射模块

#### 功能说明
允许用户将物理按键映射为任意 QMK 键码，支持多层切换（0-3层）。

#### 关键数据结构

```javascript
// 键码映射表（QMK标准）
const keyCodeToText = {
    4: 'A', 5: 'B', ...,                    // 字母键
    30: '1', 31: '2', ...,                  // 数字键
    58: 'F1', 59: 'F2', ..., 69: 'F12',     // 功能键
    224: 'LCtrl', 225: 'LShift', ...,       // 修饰键
    21024: 'MO(0)', 21025: 'MO(1)', ...,    // 换层键（Momentary）
    21056: 'TO(0)', 21057: 'TO(1)', ...,    // 切换层键（Toggle）
    30464: 'M0', 30465: 'M1', ...,          // 宏键（VIA格式 0x7700-0x770F）
    ...
};
```

#### 核心函数

| 函数名 | 功能 | 参数 | 返回值 |
|--------|------|------|--------|
| `sendVIAReport(commandId, data)` | 发送VIA命令 | `commandId`: 命令ID, `data`: 数据数组 | Promise |
| `readKey(layer, keyIndex)` | 读取指定位置键码 | `layer`: 层号, `keyIndex`: 键索引 | 键码数值 |
| `writeKey(layer, keyIndex, keyCode)` | 写入键码 | `layer`, `keyIndex`, `keyCode` | void |

#### 关键代码位置
- **VIA协议命令定义**: 行 ~1670-1690
- **键码映射表**: 行 ~1700-1763
- **键位读写函数**: 行 ~2670-2700

### 3.2 宏功能模块

#### 功能说明
支持 M0-M15 共16个宏，每个宏可设置文本内容，写入键盘后由固件执行。

#### 宏写入流程（VIA官方协议）

```
1. MACRO_GET_COUNT (0x0C)      - 获取宏数量
2. MACRO_GET_BUFFER_SIZE (0x0D) - 获取缓冲区大小
3. MACRO_RESET (0x10)          - 重置宏缓冲区
4. MACRO_SET_BUFFER (0x0F)     - 写入宏数据（分块，每块28字节）
   - offset=0xFF, size=1, data=[0xFF]  // 标记写入中
   - offset=0, size=28, data=[宏数据]   // 写入实际数据
   - ... (循环写入所有块)
   - offset=0xFF, size=1, data=[0x00]  // 标记写入完成
5. EEPROM_SAVE (0x12)          - 保存到硬件（可选，写入RAM立即生效）
```

#### 关键数据结构

```javascript
const VIA_COMMAND = {
    // ... 其他命令
    MACRO_GET_COUNT: 0x0C,
    MACRO_GET_BUFFER_SIZE: 0x0D,
    MACRO_GET_BUFFER: 0x0E,
    MACRO_SET_BUFFER: 0x0F,
    MACRO_RESET: 0x10,        // 宏缓冲区重置
    EEPROM_RESET: 0x11,
    EEPROM_SAVE: 0x12         // 保存到EEPROM
};

const MACRO_COUNT = 16;        // 宏数量
let macros = Array(MACRO_COUNT).fill(''); // 宏内容数组
```

#### 宏数据格式
- **纯文本宏**: 直接写入ASCII码，宏之间用 `0x00` 分隔
- **示例**: "HELLO" → `[0x48, 0x45, 0x4C, 0x4C, 0x4F, 0x00, ...]`

#### 核心函数

| 函数名 | 功能 | 位置 |
|--------|------|------|
| `writeMacrosToKeyboard()` | 写入宏到键盘 | 行 ~3657 |
| `saveCurrentMacro()` | 保存当前编辑的宏到数组 | 行 ~2450 |
| `loadMacros()` | 从localStorage加载宏 | 行 ~2400 |
| `saveMacros()` | 保存宏到localStorage | 行 ~2410 |

#### 关键代码位置
- **宏相关变量定义**: 行 ~1687-1690
- **VIA命令定义**: 行 ~1676-1690

### 3.4 EEPROM模块

#### 功能说明
读取和写入键盘EEPROM数据，支持查看原始十六进制数据。

#### 核心函数

| 函数名 | 功能 | 参数 |
|--------|------|------|
| `readEEPROM(offset, length)` | 读取EEPROM | `offset`: 起始地址, `length`: 长度 |
| `writeEEPROM(offset, data)` | 写入EEPROM | `offset`: 起始地址, `data`: 数据数组 |

#### EEPROM地址映射（VIA标准）

| 地址范围 | 用途 |
|----------|------|
| 0x0000-0x000F | 键盘信息 |
| 0x0010-0x004F | 键位映射（层0） |
| 0x0050-0x008F | 键位映射（层1） |
| ... | ... |
| 0x0200-0x03FF | 宏缓冲区 |

## 4. 通信协议

### 4.1 WebHID通信

```javascript
// 设备打开
await hidDevice.open();

// 发送报告（VIA协议）
await hidDevice.sendReport(reportId, data);

// 接收报告
hidDevice.addEventListener('inputreport', (event) => {
    const data = new Uint8Array(event.data.buffer);
    // 解析响应
});
```

### 4.2 VIA协议格式

**发送格式**（32字节）:
```
[commandId][data...]
```

**响应格式**（32字节）:
```
[commandId][status][data...]
```

#### 常用命令码

| 命令 | 代码 | 功能 |
|------|------|------|
| GET_PROTOCOL_VERSION | 0x01 | 获取协议版本 |
| GET_KEYBOARD_VALUE | 0x02 | 获取键盘信息 |
| SET_KEYCODE | 0x05 | 设置键码 |
| GET_KEYCODE | 0x04 | 获取键码 |
| EEPROM_READ | 0x0A | 读取EEPROM |
| EEPROM_WRITE | 0x0B | 写入EEPROM |

## 7. 常见问题与解决方案

### 7.1 连接失败

**现象**: 点击"连接设备"无响应

**排查步骤**:
1. 确认浏览器支持 WebHID（Chrome/Edge 89+）
2. 确认键盘已连接且处于VIA模式
3. 检查控制台错误信息（F12）

### 7.2 宏写入无效

**现象**: 写入宏后关闭网页无效

**可能原因**:
1. 未发送 EEPROM_SAVE（写入RAM未持久化）
2. 宏键码格式错误（需使用0x7700格式）
3. 物理按键未设置为M0-M15

**解决方案**:
```javascript
// 确保使用VIA格式的宏键码
// M0 = 0x7700 (30464), M1 = 0x7701 (30465), ...
const m0Code = 30464; // 0x7700
```

### 7.3 灯光控制无效

**现象**: 点击灯光按钮无反应

**排查步骤**:
1. 确认键盘支持灯光控制
2. 检查灯光命令码是否正确
3. 检查VIA协议版本是否支持
